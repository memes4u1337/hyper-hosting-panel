# HYPER-HOST v58

Точечное исправление Nginx runtime для серверов, где `/etc` и `/usr` доступны только для чтения.

## Изменено

- удалена запись в `/usr/lib/systemd/system`;
- удалена зависимость от `/etc/systemd/system`;
- временный unit создаётся только в `/run/systemd/system`;
- постоянный автоподъём выполняется через root `crontab @reboot`;
- рабочая конфигурация Nginx остаётся в `/opt/hyper-host/runtime/nginx`;
- установщик проверяет реальное создание, HTTP-открытие и удаление временного сайта;
- итоговый экран показывает IP, ссылки и сохранённые данные администратора без изменения пароля.
