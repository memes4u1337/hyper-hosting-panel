# HYPER-HOST v72

Точечное восстановление настоящих SSL-сертификатов после полной пересборки
Nginx-vhost'ов в v71.

## Исправлено

- vhost сайта больше не получает общий self-signed сертификат;
- сертификат подбирается по SAN/CN из существующих каталогов Let's Encrypt;
- просроченные и не подходящие сертификаты не подключаются;
- `nginx reload`, создание сайта и полное восстановление Nginx больше не
  заменяют реальный сертификат fallback-сертификатом;
- панель снова получает сертификат `panel.hyper-host.pw`, если он существует;
- добавлены команды:
  - `hyper-host-ctl nginx-restore-ssl-all`
  - `hyper-host-ctl nginx-ssl-map-json`

FTP, SQL, боты, пароль администратора и файлы сайтов не изменяются.
