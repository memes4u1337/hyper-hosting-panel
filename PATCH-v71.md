# HYPER-HOST v71

Точечное восстановление Nginx после неверной catch-all маршрутизации.

- Панель закреплена за LAN/WAN IP и `panel.hyper-host.pw`.
- Все существующие доменные каталоги автоматически получают отдельные vhost.
- `beta.mystockbot.xyz` получает собственную заглушку.
- Будущие сайты получают заглушку только при пустом `public_html`.
- Старые `hyper-host-ip-*` и конфликтующие catch-all ссылки отключаются.
- Nginx полностью перезапускается после успешного `nginx -t`, чтобы не оставались старые workers.
- Каждый домен проверяется отдельным probe-файлом через реальный LAN IP.
- FTP, SQL, боты и пароль `admin` не меняются.
