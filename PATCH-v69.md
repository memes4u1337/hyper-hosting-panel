# HYPER-HOST v69

Точечный патч поверх v68. Исправляет ложный откат v68 и старые catch-all конфиги Nginx.

## FTP/FTPS

FTP-архитектура v68 сохранена без изменений:

- LAN: `192.168.0.179:21`, PASV `40000-40049`;
- WAN backend: `192.168.0.179:2121`;
- внешний адрес: `90.189.208.25:21`, PASV `40050-40100`;
- общий набор FTP-аккаунтов;
- explicit TLS 1.2;
- внешние подключения TCP 21 перенаправляются на WAN backend.

После успешной проверки FTP компонент считается установленным. Если позже не пройдёт тест Nginx, FTP больше не откатывается.

## Nginx/sites

- конфиг панели больше не сохраняет старый `server_name _`;
- панель не может быть `default_server`;
- при настроенном `PANEL_DOMAIN` панель работает только по этому домену;
- без `PANEL_DOMAIN` панель привязывается только к LAN/WAN IP и `localhost`;
- все старые активные конфиги с `server_name _` отключаются;
- остаётся один нейтральный `00-hyper-host-default.conf`;
- точный конфиг сайта имеет `server_name <domain>` и `root <site>/public_html`;
- `index.html` имеет приоритет над `index.php`;
- HTTP и HTTPS открывают один и тот же `public_html`.

## Исправление проверки

v68 использовал `curl --resolve`, но системные proxy-переменные могли отправить запрос не в локальный Nginx. v69 проверяет:

- HTTP через `127.0.0.1` и явный заголовок `Host`;
- HTTPS через `--resolve` и `--noproxy '*'`;
- наличие точного `server_name` в активном конфиге;
- правильный `root`;
- отсутствие предупреждения `conflicting server name "_"`;
- заглушку нового сайта;
- загруженный пользователем `index.html`;
- существующий `beta.mystockbot.xyz` через probe-файл;
- нейтральный ответ для неизвестного Host.

При ошибке установщик печатает активные `server_name`, `root`, `default_server` и список включённых конфигов.

## Не изменяется

- пароль и хеш пользователя `admin`;
- файл сохранённых реквизитов администратора;
- MariaDB/MySQL;
- боты;
- содержимое существующих сайтов;
- FTP-логины и пароли.
