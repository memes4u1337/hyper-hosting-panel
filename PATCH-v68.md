# HYPER-HOST v68

Точечный патч поверх v64-v67.

## FTP/FTPS

- два независимых ProFTPD-инстанса с общей базой аккаунтов;
- LAN: `192.168.0.179:21`, PASV `40000-40049`;
- WAN backend: `192.168.0.179:2121`, публичный PASV-IP `90.189.208.25`, PASV `40050-40100`;
- внешние подключения, пришедшие на TCP 21, перенаправляются на WAN backend через iptables;
- explicit TLS 1.2, `NoSessionReuseRequired`;
- создание, удаление и смена паролей FTP-аккаунтов сохраняются.

## Nginx/sites

- панель больше не является default vhost;
- добавлен нейтральный default vhost для HTTP/HTTPS;
- каждый `hyper-host-site-*.conf` получает точный HTTPS vhost;
- новые сайты создают заглушку `index.html`;
- порядок индексов: `index.html index.htm index.php`;
- существующие HTTP-only сайты автоматически обновляются;
- загруженные в `public_html` файлы открываются вместо панели.

## Безопасность изменений

- не изменяется SQLite-пароль `admin`;
- не изменяется файл сохранённых реквизитов администратора;
- не изменяются MySQL/MariaDB, базы, боты и содержимое существующих сайтов;
- перед установкой создаётся резервная копия ProFTPD и writable Nginx runtime;
- при провале теста выполняется откат.

## Автотест установщика

- LAN FTPS PASV/upload/download/SHA-256;
- WAN backend PASV/upload/download/SHA-256;
- iptables redirect external TCP 21 -> 2121;
- создание временного сайта;
- HTTP/HTTPS заглушка временного сайта;
- замена `index.html` и проверка, что открывается новый файл;
- проверка указанного домена через временный probe-файл в его `public_html`;
- проверка, что неизвестный Host не открывает панель;
- неизменность хеша пароля `admin`.
