# HYPER-HOST v55 — read-only `/etc` FTP fix

Исправления:

- FTP больше никогда не пишет в `/etc/fstab`.
- FTP не зависит от `/etc/pam.d`, `/etc/passwd` и постоянных unit-файлов в `/etc/systemd/system`.
- Используется `pyftpdlib` в изолированном окружении `/opt/hyper-host/venv-ftp`.
- Два экземпляра: LAN `192.168.0.179:21`, WAN `90.189.208.25:2121`.
- Создание, смена пароля и удаление пользователей применяются сразу.
- Bind-mount выполняются только в runtime, а после перезагрузки восстанавливаются через root crontab.
- Berkeley DB оставлена только как необязательная совместимость и больше не может сорвать операцию.
- Установщик выполняет настоящий тест: создать временного пользователя, загрузить/скачать файл, удалить пользователя.
