# HYPER-HOST v59

Точечное исправление создания сайтов после ошибки:

```text
open() "/var/www/hyper-host-sites/<domain>/logs/access.log" failed (2: No such file or directory)
```

## Изменено

- каталоги `public_html` и `logs` создаются до публикации Nginx-конфига;
- `access.log` и `error.log` создаются до `nginx -t`;
- перед каждым reload автоматически восстанавливаются отсутствующие пути логов из активных конфигов;
- при ошибке новый конфиг откатывается, файлы сайта сохраняются;
- устранён повторный `server_name _` в конфиге панели;
- FTP, SQL, боты, база панели и файлы существующих сайтов не изменяются.
