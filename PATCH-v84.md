# HYPER-HOST v84 — очистка конфликтов панели и сайтов

Исправляет ошибку v83:

```text
conflicting server name "panel.hyper-host.pw"
```

## Что изменено

- удаляются все старые активные маршруты управляемых доменов независимо от имени файла;
- из старых конфигов в `/etc/nginx/conf.d` удаляются только конфликтующие `server_name`;
- остаётся один канонический vhost панели;
- каждый реальный `public_html` получает отдельный vhost;
- `beta.mystockbot.xyz` открывает только собственный `public_html`;
- повторные конфликты автоматически устраняются до перезагрузки Nginx;
- будущий `add-site` больше не создаёт второй активный symlink и не принимает домен панели как alias;
- файлы сайтов, FTP, SQL, боты и пароль admin не изменяются.
